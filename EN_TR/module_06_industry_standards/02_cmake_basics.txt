# CMakeLists.txt (Example / Ornek)
# ---------------------------------------------------------------------
# [THEORY: Why Do We Use CMake? / TEORİ: TEORİ: Neden CMake Kullanıyoruz?]
# EN:
# Compiling C++ code manually like `g++ a.cpp b.cpp -o main` works for small tasks.
# But what if you have hundreds of files, or external libraries written by someone 
# else like "nlohmann_json" or "Google_Test"? It becomes impossible!
# CMake is a PERFECT "meta-build" tool. It allows your project to be compiled with 
# the EXACT SAME STANDARD on Windows (Visual Studio), Linux (Make/Ninja), or Mac (Xcode).
#
# TR:
# C++ projelerinde her dosyayi "g++ a.cpp b.cpp -o main" seklinde derlemek kücük 
# projelerde calisir. Ancak yüzlerce dosyaniz ve baska birinin yazdigi "Google_Test" 
# gibi kütüphaneleriniz varsa bu mümkün degildir!
# CMake, projeyi her isletim sisteminde ayni standartta hatasız derlememizi 
# saglayan MUKEMMEL bir meta-derleme aracidir. Endüstri standardidir.
# ---------------------------------------------------------------------

# 1. Minimum CMake Version Required
# EN: Tells the system the lowest acceptable CMake version.
# TR: Projenin gerektirdigi minimum CMake sürümü.
cmake_minimum_required(VERSION 3.10)

# 2. Project Declaration
# EN: Setting the Project Name, Version, and the Language (CXX = C++).
# TR: Proje Adi, Versiyonu ve Dil Belirleme (CXX).
project(ModernCppMastery VERSION 1.0.0 LANGUAGES CXX)

# 3. Setting C++ Standards (Crucial!)
# EN: Forcing the compiler to use C++17 rules strictly.
# TR: Hangi C++ Standardini kullanacagimizi derleyiciye dayatiyoruz (Zorunlu kiliyoruz).
set(CMAKE_CXX_STANDARD 17)
set(CMAKE_CXX_STANDARD_REQUIRED True)

# 4. Source Files & Executable Target
# EN: "Create an application named MainApp utilizing these source files."
# TR: Kodlarimizin bulundugu baglantilari ve olusacak calistirilabilir dosyanin adini belirliyoruz.

# EN: If we downloaded a package via vcpkg or conan, we bind it here.
